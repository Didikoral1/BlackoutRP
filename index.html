# Discord-Teamverbindung über Cloudflare Workers GitHub Pages kann einen Discord-Bot nicht sicher direkt ausführen. Deshalb wird ein kostenloser Cloudflare Worker als sichere Zwischenstelle benutzt. ## 1. Discord-Bot erstellen 1. Öffne das Discord Developer Portal. 2. Erstelle eine Application und anschließend einen Bot. 3. Aktiviere beim Bot den **Server Members Intent**. 4. Lade den Bot auf euren Discord-Server ein. 5. Der Bot benötigt mindestens die Berechtigung, Mitglieder zu sehen. ## 2. Cloudflare Worker erstellen 1. Erstelle ein kostenloses Cloudflare-Konto. 2. Öffne **Workers & Pages**. 3. Erstelle einen Worker. 4. Ersetze den Beispielcode durch `worker.js`. 5. Lege folgende Variablen an: - `DISCORD_GUILD_ID` = ID eures Discord-Servers - `ALLOWED_ORIGIN` = `https://didikoral1.github.io` Lege außerdem als verschlüsseltes Secret an: - `DISCORD_BOT_TOKEN` = Token des Bots Der Bot-Token darf niemals in GitHub oder in `config.js` stehen. ## 3. Worker-Adresse eintragen Nach dem Veröffentlichen erhältst du eine Adresse wie: ```text https://blackout-discord-api.DEINNAME.workers.dev ``` Diese Adresse trägst du in der Website in `config.js` bei `discordTeamApi` ein. ## Verwendete Discord-Rollen - Projektleitung: `1524899549888385196` - Teamverwaltung: `1525185369036030032` - Support: `1525184472717459608` - Support: `1525184560449458206` ## Online-Status Die Teamliste zeigt standardmäßig nur die Mitglieder und Rollen. Für echte Online-/Abwesenheitsstatus-Anzeigen wäre zusätzlich eine dauerhafte Gateway-Verbindung über einen Bot-Dienst nötig. Der Worker allein fragt keine Live-Presences über das Discord Gateway ab.
